문제 4

문제 5

문제 7

4. 코드/판단형 문제
문제 26

다음 코드가 있다고 하자.

int x = 0;

그리고 x가 두 캐시라인에 걸쳐 배치되도록 강제로 조정하였다.
Thread 0이 x = -1을 반복 수행하고, Thread 1이 x를 반복해서 읽는다.

이때 발생할 수 있는 현상을 쓰시오.

절대로 0 또는 -1만 읽힌다
0과 -1 이외의 중간값이 읽힐 수 있다
항상 segmentation fault가 발생한다
mutex가 없어도 문제 없다
문제 27

다음 코드의 문제점을 설명하시오.

volatile bool flag[2];
volatile int victim;

void lock(int id) {
    int other = 1 - id;
    flag[id] = true;
    victim = id;
    while (flag[other] && victim == id) { }
}

힌트: 컴파일러 최적화와 CPU 메모리 모델을 구분해서 설명할 것.

문제 28

다음 설명 중 맞는 것을 모두 고르시오.

volatile은 CPU의 store buffer를 비워 준다.
atomic은 원자성과 메모리 순서 제어를 제공할 수 있다.
mutex는 상호배제를 제공한다.
weak memory model에서는 write 후 read의 관측 순서가 바뀔 수 있다.
캐시 coherence가 있으면 torn read는 절대 발생하지 않는다.

다음 상황에서 왜 두 쓰레드가 동시에 임계영역에 진입할 수 있는지 설명하시오.
